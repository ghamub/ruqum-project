# 𐪖𐪑𐪔𐪗 رُقم — النقوش الثمودية القديمة

> منصة تفاعلية لاستكشاف الخط الثمودي القديم، مدعومة بتقنية **نهى 2.0** من شركة **علم**

---

## 📌 وصف المشروع

**رُقم** هو تطبيق ويب تفاعلي يتيح للمستخدم:

- ✍️ **كتابة اسمه** بالخط الثمودي القديم لحظياً
- 🪨 **فكّ النقوش** — تجربة تحاكي اكتشاف نقش على صخرة حقيقية، تختار الرموز وتبني نقشك ثم يُترجم لك
- 🤖 **محادثة ذكية** مع مساعد متخصص بالنقوش الثمودية مدعوم بـ **نهى 2.0**

المشروع مبني بـ **HTML + CSS + JavaScript** للواجهة، و**Python Flask** للـ Backend الذي يخفي مفتاح الـ API بأمان.

---

## 🗂️ هيكل المشروع

```
ruqm-project/
├── app.py            ← سيرفر Flask (يحتوي منطق API)
├── index.html        ← واجهة الموقع الكاملة
├── .env              ← مفاتيح API (لا تُرفع على GitHub)
├── .env.example      ← مثال للمتغيرات البيئية
├── .gitignore        ← يحمي ملف .env
├── requirements.txt  ← مكتبات Python
└── README.md         ← هذا الملف
```

---

## ⚙️ متطلبات التشغيل

- Python 3.8 أو أحدث
- مفتاح API من منصة **علم** للوصول لموديل `nuha-2.0`

---

## 🚀 طريقة التشغيل

### 1. استنسخ المشروع
```bash
git clone https://github.com/username/ruqm-project.git
cd ruqm-project
```

### 2. أنشئ بيئة افتراضية (اختياري لكن موصى به)
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 3. ثبّت المكتبات
```bash
pip install -r requirements.txt
```

### 4. أعدّ ملف `.env`
انسخ ملف المثال الموجود في المشروع:
```bash
cp .env.example .env
```
ثم افتح `.env` وضع مفتاح API الخاص بك:
```env
NAHYA_API_KEY=ضع_مفتاحك_هنا
NAHYA_BASE_URL=https://elmodels.ngrok.app/v1
NAHYA_MODEL=nuha-2.0
```

### 5. شغّل السيرفر
```bash
python app.py
```

### 6. افتح الموقع
```
http://localhost:5000
```

---

## 🔒 الأمان

- مفتاح الـ API محفوظ في `.env` فقط على جهازك
- الـ Frontend لا يحتوي على أي مفتاح
- ملف `.env` مضاف في `.gitignore` لحمايته من الرفع على GitHub
